FalveyAPI FalveyCommitInsuranceOperation Link to Docs: Attached to "Falvey Insurance" Account in Corporate Org ("Falvey Shippers Insurance API Documentation_20230609")
Notes: The Commit web-service allows the client to commit to purchasing insurance for the merchandise that was entered.
Once this commit web-service is initiated Falvey takes in the data, creates a policy (if needed), and returns a link to download the certificate of insurance.
The Commit web-service will search the node for the existence of any of the excluded item words (as returned by the Excluded Item web service function).
If any items are found (even partial matches, i.e. if "auto" is an excluded item, "automobile" will be returned as a match)
the node must contain a valid time stamp, AND the node must contain the ExcludedItemID(s) of the matching item(s).
If an excluded item is detected by the Commit function and those two nodes are not properly supplied and populated, an error XML response will be given and the Commit will not continue.
No certificate will be issued for the current transaction. If an existing (non-voided) shipment is found which matches the supplied , the shipment will be rejected with an error message.
Duplicate ShipmentUUIDs are not allowed per CompanyUUID.
Name: FAPI - Commit
 https://www.falveycargo.com/components/ws_Commit.cfc?method=getFreightData POST

Notes: Creates or updates a load based on the given external_id. Returns the details of the created or updated load.
Modeling Loads and Shipments
A load is a truck movement, made by one carrier, carrying one or more shipments. It may have one or more picks and one or more drops.
Charges accrued against the load may be modeled as a charge property on the load, or as a set of line items. Line items are recommended over "total charge", because you can more specifically see where the expected charge differs from the actual charge.
Line item charges are also necessary in the case where multiple carriers may invoice you for a given load.
When to send load data
Send load and shipment data as early in their as possible. At a minimum we need the load data prior to the time when any documents (via email or api) would be received from carriers.
